# ProcessMonitor

Программа мониторинга и контроля работоспособности пользовательских приложений (процессов).

## Описание

ProcessMonitor — Windows-приложение, разработанное с использованием Qt 5 и C++ в среде Visual Studio. Позволяет отслеживать состояние процессов, автоматически перезапускать завершившиеся приложения, а также управлять списком отслеживаемых программ.

## Требования к системе

- Операционная система: Windows 7 и выше
- Среда разработки: Visual Studio
- Фреймворк: Qt 5.15.2

## Сборка

```bash
msbuild ProcessMonitor.sln /p:Configuration=Release /p:Platform=x64 /t:Rebuild
```

Исполняемый файл: `x64\Release\ProcessMonitor.exe`

## Реализованный функционал

### 1. Мониторинг пользовательских приложений

Реализовано в классе `ProcessMonitor`:
- Функция `isProcessRunning()` — поиск процесса по имени исполняемого файла
- Функция `getProcessStatus()` — определение статуса ("Running", "Not Responding")
- Таймер мониторинга срабатывает каждые 2 секунды

### 2. Непрерывный мониторинг

- Мониторинг осуществляется на протяжении всего времени работы программы
- Реализован через `QTimer` с интервалом 2 секунды в `MainWindow::onMonitorTimeout()`
- Проверяется состояние всех процессов из списка пользователя

### 3. Отображение списка работающих приложений

- Таблица `QTableWidget` в главном окне
- Столбцы: Application, Arguments, Delay (sec), Status, PID
- Автоматическое обновление каждые 2 секунды
- Контекстное меню (правый клик мыши) с пунктами:
  - Edit (Редактировать)
  - Delete (Удалить)
  - Terminate (Завершить)
  - Launch (Запустить)

### 4. Остановка (завершение) выбранного приложения

- Пункт меню Actions → Terminate
- Кнопка на панели инструментов
- Контекстное меню таблицы
- Использует `TerminateProcess()` WinAPI
- После завершения `autoRestart` отключается

### 5. Ведение списка приложений пользователем

**Добавление:**
- Меню: Edit → Add
- Панель инструментов: кнопка "+"
- Диалоговое окно с полями:
  - Program path (с обзором файлов)
  - Arguments (аргументы командной строки)
  - Delay on start (задержка в секундах)

**Редактирование:**
- Меню: Edit → Edit
- Контекстное меню: Edit
- Позволяет изменить путь, аргументы и задержку

**Удаление:**
- Меню: Edit → Delete
- Контекстное меню: Delete
- При запущенном приложении — запрос подтверждения

### 6. Автоматический запуск приложений из списка пользователя

- При запуске мониторинга проверяется `delaySeconds`
- Если `delaySeconds == 0` — запуск немедленный
- Если `delaySeconds > 0` — запуск через указанный интервал
- Реализовано в `scheduleDelayedLaunch()`

### 7. Автоматический перезапуск приложения

- Если приложение закрылось или аварийно завершилось
- `autoRestart = true` по умолчанию
- В `onMonitorTimeout()` при `pid == 0` и `autoRestart == true` выполняется перезапуск
- Отслеживается через `updateStatus()`

### 8. Отложенный запуск

- Для каждого приложения можно задать индивидуальную задержку
- Задержка отсчитывается от момента запуска мониторинга
- Реализовано через `QTimer` с `setSingleShot(true)`
- Разные приложения могут иметь разные интервалы

### 9. Логирование

- Лог-файл: `monitor.log` в каталоге программы
- Формат: `yyyy-MM-dd hh:mm:ss - сообщение`
- Логируемые события:
  - Запуск приложения (Launched)
  - Завершение приложения (Terminated by user)
  - Ошибки запуска/завершения
  - Изменение статуса (Running, Not Responding, Stopped)
  - Обнаружение процесса
- Реализовано в классе `Logger` (потокобезопасный)
